版本控制工具

  • 集中式版本控制工具
    • CVS、SVN、VSS
  • 分布式版本控制工具
    • Git、Mercurial、Bazaar、Darcs…….

Git的优势

  • 大部分在本地操作完成，不需要联网
  • 完整性保证
  • 尽可能添加数据而不是删除或修改数据
  • 分支操作非常快捷流畅
  • 与Linux命令全面兼容

Git结构

  • 本地库
  • 历史版本
  • 暂存区
  • 临时版本
  • 工作区
  • 写代码
  • 工作区（git add）——>暂存区（git commit）——->本地库

Git命令行操作

本地库操作

本地库初始化

  • 命令：git init
  • 效果:
  • 注意：.git目录中存放的是本地库相关的子目录和文件，不要删除，也不要胡乱修改

设置签名

  • 形式

    • 用户名：daidai
    • Email地址：@@@@@@
    • 作用：区分不同开发人员的身份
    • 辨析：这里设置的签名和登陆远程库（代码托管中心）的账号、密码没有任何关系
    • 命令：
      • 项目级别/仓库级别：仅在当前本地库范围内有效
        • git config user.name daidai_pro
        • git config user.email daidai_pro@qq.com
        • 信息保存的位置：在.git目录下的config文件里
      • 系统用户级别：登陆当前操作系统用户的范围
        • git config –global user.name daidai_glb
        • git config –global user.email daidai_pro@qq.com
        • 信息保存位置：在系统用户目录下的.gitconfig文件中
      • 级别优先级：
        • 就近原则：项目级别优先于系统用户级别，二者都有时，采用项目级别的签名
        • 如果只是系统用户级别的签名，就以系统用户级别签名为准
        • 二者都没有是不允许的

基本操作

  • 状态查看操作

    • git status

    查看工作区和暂存区的状态

  • 添加操作

    • git add [file name]

    将工作区的“新建/修改”添加到暂存区域

  • 提交操作

    • git commit -m “commit message” [file name]

    将暂存区的内容提交到本地库

  • 查看历史记录操作
  • git log
    • 多屏显示的控制方法
      • 空格—向下翻页
      • b——-向上翻页
      • q——-退出
    • git log –pretty=oneline
    • git log –oneline
    • git reflog
    • HWAD@{移动到当前版本所需要的步数}
  • 前进后退
    • 基于索引值的操作（推荐使用）
      • git reset –hard [局部索引值]
      • git reset –hard 94eb597
    • 使用^符号（只能后退，不能往前）
      • git reset –hard HEAD^（一个^表示后退一步，n个^表示后退n步）
    • 使用~符号（只能后退）
      • git reset –hard HEAD~n（后退n步）

  • reset命令的三个参数对比

    • –soft
      • 仅仅在本地库移动HEAD指针
    • –mixed
      • 在本地库移动HEAD指针
      • 重置暂存区
    • –hard

      • 在本地库移动HEAD指针

      • 重置暂存区

      • 重置工作区

  • 删除文件并找回

    • 前提：删除前，文件存在是的状态提交到了本地库
    • 操作：git reset –hard [指针位置]
      • 删除操作已经提交到本地库：指针位置指向历史记录
      • 删除操作尚未提交到本地库：指针位置使用HEAD
  • 比较文件差异
    • git diff [文件名]
      • 将工作区中的文件和暂存区进行比较
    • git diff [本地库中的历史版本] [文件名]
      • 将工作区中的文件和本地库历史记录比较
    • 不带文件名将会比较多个文件

分支管理

  • 什么是分支？

  • 在版本控制过程中，使用多条线同时推进多个任务

  • 分支的好处

    • 同时并行推进多个功能的开发，提高开发效率
    • 各个分支在开发过程中，如果某一个分支开发失败，不会对其他分支有任何影响，失败的部分删除重新开始即可。

  • 分支的操作

    • 创建分支

      • git branch [分支名]

    • 查看分支

      • git branch -v

    • 切换分支

      • git checkout [分支名]

    • 合并分支

      • 第一步：切换到接受修改分支上（被合并，增加新内容）

        git checkout [被合并的分支名]

      • 第二步：执行merge命令

        git merge [有新内容的分支名]

    • 解决冲突

      • 冲突的表现

      • 冲突的解决

        • 第一步：编辑文件，删除特殊符号
        • 第二步：把文件修改到满意的程度为止，保存退出
        • 第三步：git add [文件名]
        • 第四步：git commit -m [日志信息]
          • 注意：此时的commit一定不能带具体文件名

远程库操作

  • 在git中保存远程库地址

    • 查看当前远程库地址列表
      • git remote -v
    • 创建远程库地址别名
      • git remote add [别名] [远程库地址]
      • git remote add origin https://github.com/Benson-QAQ/git2.git
    • 效果

  • 推送

    • 命令
      • git push [远程库地址] [需要推送的分支]
      • git push origin master
      • 注意：分支必须写

  • 克隆

    • 命令
      • git clone [远程库地址]
    • 效果
      • 完整的把远程库下载到本地
      • 创建origin远程地址别名
      • 初始化本地库

  • 拉取

    • pull

      • git pull [远程库地址别名] [远程分支名]

    • pull = fetch + merge

      • git fetch [远程库地址别名] [远程分支名]
      • git merge [远程地址别名/远程分支名]

  • 解决冲突

    • 要点
      • 如果不是基于GitHub远程库的最新版所做的修改，不能推送，必须先拉取。
      • 拉取下来后如果进入冲突状态，则按照“分支冲突解决”操作解决即可

fork

  • 以第三者的账号区访问远程库地址，然后点击fork

SSH免密登陆

当只操作一个账号时候使用ssh更加方便

  • 删除当前ssh记录
    • rm -r .ssh/
  • 生成ssh目录
    • ssh-keygen -t rsa -C [邮箱地址]
  • 在ssh目录的.pub文件下复制密钥，在github上进行添加密钥

git config –system –unset credential.helper

​

​

git config –global –unset credential.helper

git config –local –unset credential.helper
